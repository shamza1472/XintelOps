# XIntelOps Posting Cadence

Manual posting only — copy from the email brief. Never auto-post.

## Every scan (8× daily, every 3 hours PKT)

| Slot | Platform | What to post | When |
|---|---|---|---|
| **Primary** | X | Top verified signal (`x_post` or thread) | Within **30 minutes** of email |
| **Secondary** | X | "What most people missed" | **4–6 hours** after primary, or at next scan |
| **Engagement** | X | Journalist comment (1/day max) | When you find their relevant post |

## LinkedIn (Mon / Wed / Fri only)

| Day | Action |
|---|---|
| Monday, Wednesday, Friday | Post `linkedin_post` between **09:00–11:00 PKT** |
| Other days | Skip LinkedIn — draft is saved for the next window |

## Crisis mode

When `crisis_detected=true`:

1. Post the X brief **immediately** (within 15 minutes)
2. Skip secondary/engagement until the primary is live
3. LinkedIn: only if Mon/Wed/Fri — post a shorter crisis analysis (still 200+ words)

## Weekly rhythm (authority building)

| Content type | Target frequency | Purpose |
|---|---|---|
| Breaking signal posts | 1–2/day | Visibility, timeliness |
| "What most missed" | 1/day | Differentiation |
| Journalist comments | 1/day | Network positioning |
| LinkedIn flagship | 3/week | Institutional credibility |

## Source discipline

Every post should trace to the **Source Verification** block in the email:

- Cite the event date (must be within 24h)
- Keep source URLs handy if challenged
- Do not post if `confidence` is not HIGH unless you add explicit caveats
